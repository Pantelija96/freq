1. Service Overview

The backend provides REST APIs, dashboard authentication, device provisioning, dashboard WebSocket events, PDF reporting, and frequency analytics built from ingested device data.

Core backend responsibilities:
  • Register and authenticate devices
  • Maintain live dashboard and device session state
  • Store commands, crashes, statistics, and frequency segments
  • Generate licences and PDF reports

2. Authentication and Roles

Area Description
Dashboard login Uses `/api/auth/login` and returns a JWT token for frontend sessions.
Admin role Can view all pages and send commands to devices.
User role Can view all data but cannot dispatch commands.
Device login Returns the device identifier and WebSocket address used for the live tunnel.
The backend currently supports local dashboard authentication, while remaining structured for future LDAP or Active Directory integration.

3. Device Provisioning and Live Flow

Devices use the open provisioning flow intended for the internal cruise-ship environment. Provisioning creates or updates device registration data and enables the device to proceed into login and WebSocket authentication.

Step Result
`/api/devices/provision` Registers the device and returns the provisioning result.
`/api/devices/login` Returns device login result and `WSS_URL` for live connection.
WebSocket `auth` Marks the device online and starts live dashboard updates.
Device logout or disconnect Marks the device offline and broadcasts the session change.

4. Analytics and Reporting

Frequency data is ingested as segments and analyzed into fixed sessions. The backend correlates crashes against these sessions and surfaces summary values to both the frontend and the generated device report PDF.

  • Total observed time
  • Total fixed time
  • Fixed session count
  • Crashes during fixed sessions
  • Crashes outside fixed sessions
  • Top fixed frequencies for small and big cores
Reports are generated one device per page and include a frequency summary, fixed-session rows, and crash history.
