1. Login and Navigation

The frontend opens on the login page. After valid credentials are entered, the user is redirected to the dashboard. The main navigation remains available from the left sidebar on desktop, or from the burger menu on smaller screens.

Admin users can view all pages and send commands to devices. User-role accounts can view all screens but cannot send commands.
Area Description
Top navigation bar Shows branding, current signed-in user, and logout action.
Sidebar menu Provides access to Dashboard, Licences, Report, and Statistics pages.
Breadcrumbs Display the current page hierarchy for easier orientation.
Figure 1. Login page and responsive navigation layout.

2. Home Dashboard

The home page provides the operational summary of the monitored fleet. The top cards display total devices, online devices, and total groups. Below the summary cards, the interface shows live charts for active devices per group and the current group distribution.

  • The dashboard listens for live backend updates through WebSocket.
  • The device table supports search, sorting, paging, and group-level display.
  • Admins can send commands directly from the dashboard table.
Figure 2. Dashboard with summary cards, live charts, and grouped device table.

3. One Device Statistics

The statistics page provides a detailed live view for a selected device. If the page is opened without a device identifier, a searchable device list is shown first. Selecting a device opens the full statistics layout.

Section Description
CPU Frequency Timeline Displays the small-core and big-core frequency lines in real time.
Fixed Session Summary Shows observed time, fixed time, fixed percentage, and most-used fixed frequencies.
Crash Correlation Separates crashes during fixed sessions from crashes outside fixed sessions.
Data tables Provide app statistics, crash history, command history, and fixed-session rows.
Figure 3. One-device statistics with fixed-session analytics and crash correlation.

4. Licences and Reports

The Licences page displays stored licence keys for registered devices. The Report page allows one or more devices to be selected and exported into a PDF report. Each device is rendered on its own page inside the generated file.

  • Licence keys are generated from the device identifier and IMEI.
  • The report PDF includes frequency summary, fixed sessions, and application crashes.
  • Crash correlation between fixed and non-fixed periods is included in the report summary.
Figure 4. Report selection page and generated PDF export flow.

5. Quick Reference

User action Result
Sign in on the login page Opens the dashboard after successful authentication.
Use the sidebar or burger menu Navigates to Dashboard, Licences, Report, or Statistics pages.
Open Statistics for a device Shows live frequency, fixed-session analytics, crashes, and command history.
Generate report Downloads a multi-page PDF with one page per selected device.
Use an admin account Enables command dispatch from dashboard and device views.
